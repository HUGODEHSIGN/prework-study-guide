HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Footer includes what's at the bottom of the page, usually more information about the page such as contacts and other links.

CSS

  • use selectors to select what elements need to be changed
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • border creates an outline around the element's box space.
  • box-shadow creates a drop shadow around the element's box

Git

  • prefix all git commands with 'git'
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • you must have a remote and optionally, ssh key, set up first to push to github
  • pushing to git goes as so:

    1. have ssh key and remote set up so git knows which repository to push to

    2. git add -A

      adds all of the files not in gitignore into a pool

    3. git commit -m 'your commit name here'

      readies the pool and allows you to add add a name to your commit for tracking versions

    4. git pull 'remote' 'branch'

      pulls the code from the specified branch, usually the main to make sure there are no problems. This step technically isn't required, but prevents issues down the line.

    5. git push 'remote' 'branch'

      files are uploaded to github here, with a chance to specify the remote and branch
